vp

Standard Syntax:
     <%@ taglib prefix="vp" uri="http://www.veriplace.com/tags" %>
XML Syntax:
     <anyxmlelement xmlns:vp="http://www.veriplace.com/tags" />
 
Custom tags for using the Veriplace web framework from JSP.

Tag Library Information
Display NameNone
Version1.0
Short Namevp
URIhttp://www.veriplace.com/tags
 

Tag Summary
setup

Makes Veriplace APIs available to the current page. Place this tag on any page that uses Veriplace functionality; it must appear before any other Veriplace custom tags.

require-userTriggers a user discovery request, if the current Veriplace user is not known yet. The rest of the page will not execute until a valid user ID is available; will redirect to an error page if the request fails.
require-locationRequests the current user's location from Veriplace. The rest of the page will not execute until a valid location is available; will redirect to an error page if the request fails.
require-location-permissionRequests permission for Veriplace to locate the current user, but doesn't actually acquire a location. The rest of the page will not execute until permission has been obtained; will redirect to an error page if the request fails. Use this tag if you want to identify the user and obtain permission ahead of time, when the actual location request will be made by a different page. The tag returns the current Veriplace user object.
require-set-locationRequests a permission token from Veriplace for changing the current user's location (not currently supported). In a future release, this capability will be selectively enabled for applications with additional security constraints.
require-location-updateUpdate the current user's location in Veriplace. The rest of the page will not execute until location is successfully updated; will redirect to an error page if the request fails.
longitudeOutputs the longitude of the current location. This is equivalent to <%= location.getLongitude() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
latitudeOutputs the latitude of the current location. This is equivalent to <%= location.getLatitude() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
accuracyOutputs the uncertainty of the current location. This is equivalent to <%= location.getAccuracy() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
uncertaintyOutputs the uncertainty of the current location. This is equivalent to <%= location.getAccuracy() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
streetOutputs the street address of the current location. This is equivalent to <%= location.getStreet() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
neighborhoodOutputs the neighborhood name of the current location. This is equivalent to <%= location.getNeighborhood() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
cityOutputs the city name of the current location. This is equivalent to <%= location.getCity() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
stateOutputs the state name of the current location. This is equivalent to <%= location.getState() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
postal-codeOutputs the postal code of the current location. This is equivalent to <%= location.getPostal() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
country-codeOutputs the country code of the current location. This is equivalent to <%= location.getCountryCode() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
location-errorOutputs the error/status message, if any, from the current location. This is equivalent to <%= location.getMessage() %>, but doesn't require you to have the location in a named variable, and won't cause an error if there is no location.
address-lineOutputs the street address (if any), neighborhood (if any), city, state, and postal code of the current location. Does nothing if there is no location.
user-fieldsGenerates hidden form field(s) containing the current Veriplace user ID, if any. Use this inside a form if the current page has already gone through user and/or location discovery, to pass on the user identity to the page that handles the form request; otherwise, that page will have to repeat the user discovery process.
